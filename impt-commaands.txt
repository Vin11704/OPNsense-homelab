- set vbox network to NAT
Change VirtualBox Settings:

    Go to VirtualBox -> Proxmox VM Settings -> Network.

    Change "Attached to:" back to NAT.

Reconfigure Proxmox:

    Because Proxmox is no longer on your 192.168.0.x network, it needs a new IP. VirtualBox NAT usually uses the 10.0.2.x subnet.

    On the Proxmox Host shell, edit the interfaces file: nano /etc/network/interfaces

    Change the vmbr0 address to 10.0.2.15/24 and gateway to 10.0.2.2.

Reconfigure Ubuntu:

    Your Ubuntu VM also needs to change. Edit its Netplan file to use an IP like 10.0.2.50/24 with the gateway 10.0.2.15 (pointing to Proxmox).

the reason why bridged adapter does not work with nested VM is that at layer 2, the packets leave the VM with a unique, virtual MAC address. which is	 rejected by the home router's wireless protocol. MAC address wireless auth token must match that of my laptop. This is also why changing routing rules in layer 3 (IP or routing) will not work. With NAT, vbox takes the data out of the packet, puts it in a windows network request, and sends using the actual host IP and MAC address.
